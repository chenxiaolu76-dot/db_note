## 模块间逻辑关系（新增，可直接用于 XMind 关系线）
- 总主线（从“硬件约束”到“系统优化”）
  - 模块一 底层存储硬件基础 -> 模块二 存储层级与缓存核心原理
    - 逻辑：硬件速度/容量/成本差异，直接决定为什么必须分层与缓存
  - 模块二 存储层级与缓存核心原理 -> 模块三 磁盘I/O原理与访问优化
    - 逻辑：缓存命中/失效行为，最终体现在磁盘I/O次数和访问模式上
  - 模块三 磁盘I/O原理与访问优化 -> 模块四 数据库记录与文件组织
    - 逻辑：I/O代价模型决定记录怎么放、页怎么设计、文件怎么组织
  - 模块四 数据库记录与文件组织 -> 模块五 数据存储布局优化
    - 逻辑：先有基本组织（堆/顺序/B+树/哈希），再做分区与行列式等高级优化
  - 模块四 数据库记录与文件组织 -> 模块六 缓冲区管理
    - 逻辑：缓冲池管理的对象就是“页”；RID/槽式页是缓存命中后的读取入口
  - 模块六 缓冲区管理 -> 模块七 内存数据库存储优化
    - 逻辑：理解传统缓冲池与页机制后，才能看懂内存数据库在“去页间接寻址/去页锁”上的优化
- 关键交叉关系（建议在 XMind 增加横向连线）
  - 模块一 -> 模块三
    - 逻辑：HDD/SSD物理特性差异，决定I/O优化手段重点（寻道优化 vs 并行/写放大控制）
  - 模块二 -> 模块六
    - 逻辑：缓存理论在数据库中的工程落地就是缓冲池、替换策略、Pin/Unpin
  - 模块五 -> 模块六
    - 逻辑：分区策略、行列存储会改变访问局部性，进而影响缓冲池命中率与替换效果
  - 模块五 -> 模块七
    - 逻辑：列式/冷热分层等布局思想在内存数据库中继续存在，但实现手段转向CPU友好与内存友好
  - 模块三 <-> 模块六
    - 逻辑：缓冲池减少真实磁盘I/O；而I/O成本又反向约束缓冲策略设计（形成闭环）
- 学习顺序建议（最稳路径）
  - 第1阶段：模块一 -> 模块二 -> 模块三（先建立性能瓶颈认知）
  - 第2阶段：模块四 -> 模块六（掌握数据库页与缓冲管理主干）
  - 第3阶段：模块五 -> 模块七（进入高级布局与内存数据库优化）
- 每个模块的“输入 -> 输出”（用于在 XMind 中补充备注）
  - 模块一 输入：硬件知识 输出：性能边界与介质特性模型

  - 模块三 输入：缓存与介质模型 输出：I/O时间分解与访问优化手段
  - 模块四 输入：I/O成本模型 输出：记录格式、页结构、文件组织设计
  - 模块五 输入：基础文件组织 输出：面向查询负载的布局优化方案
  - 模块六 输入：页与文件组织 输出：缓冲池调度、并发与替换策略
  - 模块七 输入：传统磁盘数据库机制 输出：内存数据库去瓶颈优化范式
## 模块一 底层存储硬件基础
- 易失性存储（RAM）
  - SRAM 静态随机存储器
    - 电路结构：6-8个晶体管/bit
    - 核心特性：无需刷新、访问速度快、成本高
    - 应用场景：CPU各级缓存
  - DRAM 动态随机存储器
    - 电路结构：1个晶体管+1个电容/bit
    - 核心特性：需周期性刷新、访问速度慢、成本低
    - 应用场景：主存、帧缓存
  - 增强型DRAM
    - SDRAM 同步DRAM
    - DDR SDRAM 双倍数据率DRAM（DDR/DDR2/DDR3/DDR4）
- 非易失性存储
  - 机械磁盘HDD
    - 物理结构：盘片、磁道、扇区、柱面、磁头、主轴
    - 访问时间：寻道时间+旋转延迟+数据传输时间
  - 固态硬盘SSD
    - 核心结构：闪存转换层FTL、闪存块/页、DRAM缓冲区
    - 核心特性：页读写、块擦除、有擦写寿命、磨损均衡
    - 与HDD优劣势对比
  - 其他非易失性存储：ROM、EEPROM、3D XPoint
- 磁盘接口与网络存储
  - 接口标准：SCSI、SATA、SAS、NVMe
  - 网络存储：SAN 存储区域网络、NAS 网络附加存储
- RAID 磁盘阵列技术
  - 核心目标：并行提升性能、冗余提升可靠性
  - 标准RAID级别
    - RAID0：条带化、无冗余、性能最高、无容错
    - RAID1：镜像、100%冗余、容错能力强、空间利用率50%
    - RAID3：字节级条带+专用校验盘、容错1块盘
    - RAID4：块级条带+专用校验盘、容错1块盘
    - RAID5：块级条带+分布式校验、容错1块盘、工业界主流
    - RAID6：块级条带+双分布式校验、容错2块盘
  - 组合RAID级别：RAID10、RAID50、RAID60
  - 实现方式：软件RAID、硬件RAID
## 模块二 存储层级与缓存核心原理
- 存储层级金字塔
  - 层级排序（从上到下：容量更小、速度更快、单位成本更高）
    - L0：CPU寄存器
    - L1-L3：SRAM缓存
    - L4：DRAM主存
    - L5：本地二级存储（HDD/SSD）
    - L6：远程二级存储（SAN/NAS）
  - 核心规律：越上层速度越快、容量越小、单位字节成本越高；越下层反之
- 缓存核心机制
  - 缓存定义：更小更快的存储设备，作为更大更慢设备的子集暂存区
  - 核心理论基础：局部性原理（时间局部性、空间局部性）
  - 核心概念
    - 块：缓存与下层存储的数据传输单元
    - 命中：请求数据已在缓存中
    - 失效：请求数据不在缓存中
    - 放置策略：数据块放入缓存的哪个位置
    - 替换策略：缓存满时驱逐哪个数据块
  - 核心理想效果：用底层存储的低成本，实现接近顶层存储的访问速度
- 全系统缓存实例
  - 硬件管理：寄存器、TLB、L1-L3缓存
  - 硬件+OS管理：虚拟内存、缓冲区缓存
  - 固件/软件管理：磁盘缓存、网络缓存、浏览器缓存、Web代理缓存
## 模块三 磁盘I/O原理与访问优化
- 磁盘访问时间拆解
  - 寻道时间：磁臂移动到目标磁道的时间
  - 旋转延迟：盘片旋转到目标扇区的时间
  - 数据传输时间：数据读写的实际传输时间
- 完整磁盘I/O流程
  - 步骤1：CPU发起磁盘读请求，写入命令、逻辑块号、目标内存地址
  - 步骤2：磁盘控制器读取目标扇区，执行DMA直接内存访问，将数据写入主存
  - 步骤3：DMA传输完成，磁盘控制器通过中断通知CPU
- 磁盘块访问优化手段
  - 缓冲：内存缓冲区缓存磁盘块
  - 预读：提前读取同磁道的后续块
  - 磁盘臂调度：电梯算法，最小化磁臂移动
  - 文件组织：连续块分配、区分配、碎片整理
  - 非易失性写缓冲区
## 模块四 数据库记录与文件组织
- 数据库文件基础存储模型
  - 层级映射：数据库→文件集合→记录序列→字段序列
  - 核心前提：记录大小小于磁盘块（页）大小
- 定长记录存储
  - 基础存储规则：第i条记录起始地址 = n*(i-1)，n为单条记录长度
  - 优化规则：禁止记录跨块边界
  - 记录删除的3种方案
    - 方案1：后续记录整体前移填补空位
    - 方案2：末尾记录移动填补空位
    - 方案3：空闲链表法（工业界主流）
- 变长记录存储
  - 变长记录产生场景：变长字段、多记录类型同文件、重复字段
  - 标准存储格式
    - 定长头部：空值位图、变长字段(偏移量,长度)数组
    - 变长数据区：所有变长字段实际数据
- 槽式页（Slotted Page）结构
  - 核心作用：实现记录物理位置与外部寻址解耦，支持变长记录高效管理
  - 页结构
    - 页头：记录条目数、空闲空间结束位置、槽数组
    - 槽数组：每个槽存储对应记录的「页内偏移量+记录长度」
    - 空闲空间：槽数组与记录数据区的中间区域
    - 记录数据区：从页尾反向向前存储记录
  - 核心寻址规则：外部仅通过「页号+槽号（RID）」寻址，不直接指向记录
  - 核心优势：记录可自由移动、寻址永不失效、空间利用率高、完美支持变长记录
- 大对象（LOB）存储方案
  - 基础方案：文件系统存储、数据库专属文件存储、分片存储
  - 工业级实现：PostgreSQL TOAST机制
- 记录在文件中的5种组织方式
  - 堆文件组织
    - 核心规则：记录可放在文件任意空闲位置
    - 配套机制：两级空闲空间映射表
    - 适用场景：OLTP默认表结构，配合索引使用
  - 顺序文件组织
    - 核心规则：按搜索键顺序连续存储记录
    - 增删处理：删除用指针链、插入用溢出块、定期重组
    - 适用场景：静态表、频繁顺序扫描的表
  - 多表聚簇文件组织
    - 核心规则：多个频繁关联的表存储在同一个文件/块中
    - 优劣势：关联查询性能大幅提升，单表查询性能下降
    - 适用场景：数据仓库维度表与事实表、频繁join的表组
  - B+树文件组织
    - 核心规则：记录按搜索键存储在B+树叶子节点
    - 适用场景：MySQL InnoDB索引组织表
  - 哈希文件组织
    - 核心规则：哈希值决定记录所在磁盘块
    - 优劣势：等值查询性能极高，不支持范围查询
    - 适用场景：仅等值查询的表
## 模块五 数据存储布局优化
- 表分区技术
  - 核心定义：大逻辑表拆分为多个物理独立的分区，上层应用透明
  - 核心优势
    - 分区裁剪：减少查询扫描范围，降低I/O
    - 降低维护成本：历史数据直接删除分区
    - 冷热分离：不同分区存储在不同介质
  - 典型场景：按时间分区的交易表
- 行式存储 vs 列式存储
  - 行式存储
    - 核心规则：一条记录的所有字段连续存储
    - 优势：整行增删改查性能高、支持事务ACID
    - 适用场景：OLTP事务处理系统
  - 列式存储
    - 核心规则：表的每个列单独存储
    - 核心优势：减少无效I/O、CPU缓存命中率高、压缩比高、支持SIMD向量处理
    - 核心缺点：元组重构成本高、增删改开销大、解压有CPU开销
    - 适用场景：OLAP数据仓库、大数据分析
  - 主流列式存储格式：Apache Parquet、ORC
- 数据字典（系统目录）存储
  - 核心定义：存储数据库元数据的系统表
  - 核心存储内容
    - 关系元信息：表名、字段、视图、完整性约束
    - 用户与权限信息
    - 统计信息：表记录数、字段分布
    - 物理存储信息：表组织方式、物理位置
    - 索引信息
## 模块六 缓冲区管理
- 核心定位与基础概念
  - 核心目标：最小化磁盘I/O，最大化数据内存命中
  - 核心概念
    - 磁盘块：数据库存储与传输的最小单位
    - 缓冲区：内存中存储单个磁盘块的固定大小空间
    - 缓冲池：缓冲区的集合，数据库核心内存区域
    - 缓冲区管理器：缓冲池的管理子系统，所有数据访问必经之路
- 缓冲区管理器完整工作流程
  - 步骤1：接收上层的磁盘块访问请求
  - 步骤2：通过页表检查块是否在缓冲池中
  - 步骤3：命中：直接返回块的内存地址，Pin计数+1
  - 步骤4：未命中：分配空闲缓冲区，无空闲则执行替换策略驱逐块
  - 步骤5：驱逐块时，脏块需先写回磁盘
  - 步骤6：从磁盘读取目标块到缓冲区，更新页表与元数据
  - 步骤7：返回块的内存地址给请求方
- 缓冲池并发控制机制
  - Pin/Unpin与Pin计数
    - Pin：线程访问前钉住页面，Pin计数+1
    - Unpin：访问完成后解钉，Pin计数-1
    - 核心规则：仅Pin计数=0的块可被驱逐
  - 缓冲块锁机制
    - 共享锁：读操作加锁，支持多线程并发读
    - 排他锁：写操作加锁，与共享锁互斥
- 缓冲区替换策略
  - LRU 最近最少使用
    - 规则：驱逐最久未被访问的块
    - 优劣势：通用场景命中率高，全表扫描会导致缓存污染
  - Toss-immediate 立即丢弃
    - 规则：顺序扫描的块，处理完最后一条记录立即释放
    - 适用场景：大规模顺序扫描、全表扫描
  - MRU 最近最多使用
    - 规则：驱逐最近刚被访问的块
    - 适用场景：循环访问模式的查询，如嵌套循环join
  - 混合策略+查询优化器Hint
- 面向磁盘的数据库数据组织完整流程
  - 步骤1：查询通过B+树索引，拿到记录的RID（页号+槽号）
  - 步骤2：向缓冲区管理器请求对应页号的磁盘块
  - 步骤3：缓冲区管理器通过页表检查页是否在缓冲池，未命中则从磁盘加载
  - 步骤4：拿到页的内存地址后，通过槽号从页头获取记录偏移量，读取记录
  - 步骤5：修改记录后，页标记为脏页，后台异步写回磁盘
  - 步骤6：缓冲池满时，按替换策略驱逐Pin计数=0的块
## 模块七 内存数据库存储优化
- 核心设计背景
  - 传统数据库的槽式页、缓冲池、RID寻址、页锁，都是为适配磁盘慢速、块访问特性设计
  - 内存数据库全量数据常驻内存，无需磁盘I/O，传统设计成为性能开销
- 内存数据库核心存储组织设计
  - 无需槽式页的复杂间接寻址，仍可按块/页组织元组
  - 索引直接存储记录的内存指针，而非RID
  - 定长与变长数据分离内存池存储，减少内存碎片
- 两大核心性能优化
  - 消除页间接寻址
    - 传统磁盘数据库：索引→RID→页表→内存页→槽→记录（两层间接跳转）
    - 内存数据库：索引→内存指针→记录（一步直达）
  - 规避页锁存（Latch）开销
    - 传统磁盘数据库：页访问必须加页锁，原子操作带来高CPU开销
    - 内存数据库：无缓冲池页换入换出，无需页级锁，仅需行级锁/无锁结构
- 最终效果：相比传统磁盘数据库，性能提升1-2个数量级
